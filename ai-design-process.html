Process

From brief to build.

A four-stage pipeline. Playground grounds generation in our design system. Claude Code generates. Rivet refines in place. Handoff ships.

01 — Ground Playground

Design system host.

Static site holding our full DS with labeled, rendered components. Every agent references it — output matches tokens by default.

wgiash.github.io/Playground
02 — Generate Claude Code

Grounded generation.

Agents generate flows, screens, and PRD templates against Playground. No re-describing the system every prompt.

flows · screens · PRDs
03 — Refine Rivet

Edit in place.

Overlay edit tools on the running app. Click an element, prompt the change, done. Feels like a Figma panel on real code.

click · prompt · commit
04 — Ship Handoff

Out the door.

Finished designs go to Figma or straight to engineering. The handoff shrinks as generation gets more reliable.

Figma · engineering
